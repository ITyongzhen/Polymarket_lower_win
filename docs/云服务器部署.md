# 云服务器部署

这份文档给当前项目一个最直接的上云运行流程。

## 当前价格源状态

在 Chainlink 凭证还没申请下来之前，模拟盘当前使用的是：

- `btc` `eth` `sol` `xrp` `doge` `bnb`：Binance 现货 K 线
- `hype`：Hyperliquid K 线

注意：

- 现在的模拟盘还没有把 Chainlink 报告正式接入成交判断和结算回放
- 但已经内置了“临近结算源不一致保护”，会尽量避免把 `Binance close` 误当成 `Polymarket` 真实结算价
- 等 Chainlink `API key / secret` 申请下来后，可以先开启原始报告采集，再逐步把结算代理从 Binance 切到 Chainlink

## 推荐服务器

建议先用一台最普通的 Linux 云服务器即可：

- `Ubuntu 22.04` 或 `Ubuntu 24.04`
- `2 vCPU`
- `2 GB RAM`
- 稳定公网网络

这个项目当前是 Python 常驻轮询程序，资源消耗不高。

## 1. 连接服务器

本地执行：

```bash
ssh root@你的服务器IP
```

如果你不是 `root`，把下面命令里的用户目录替换成你自己的即可。

## 2. 安装系统依赖

```bash
apt update
apt install -y git curl python3 python3-venv python3-pip
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs
npm install -g pm2
```

## 3. 校准系统时间

Chainlink WebSocket 鉴权对时间比较敏感，所以这一步很重要。

```bash
apt install -y chrony
systemctl enable chrony
systemctl restart chrony
timedatectl status
```

看到 `System clock synchronized: yes` 会更稳。

## 4. 拉取项目

推荐直接用 GitHub 仓库拉取：

```bash
cd /root
git clone https://github.com/ITyongzhen/Polymarket_lower_win.git
cd Polymarket_lower_win
```

如果你不是 `root` 用户，就把 `/root` 换成你的工作目录。

## 5. 创建虚拟环境并安装

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -U pip
pip install -e .
```

## 6. 准备环境变量

```bash
cp .env.example .env
nano .env
```

你至少要确认这些参数：

- `PM_SYMBOLS`
- `PM_TIMEFRAMES`
- `PM_BANKROLL_USD`
- `PM_SHARES_PER_SIGNAL`
- `PM_CHILD_SHARES`
- `PM_MAX_SHARES_PER_MARKET`
- `PM_LOGS_ROOT`
- `PM_CHAINLINK_API_KEY`
- `PM_CHAINLINK_API_SECRET`

如果 Chainlink 还没申请下来：

- `PM_CHAINLINK_API_KEY` 留空
- `PM_CHAINLINK_API_SECRET` 留空
- 先只跑模拟盘即可

如果你懒得手动复制，下面的一键启动脚本也会在发现 `.env` 不存在时，自动从 `.env.example` 复制一份出来。

## 7. 一条命令启动

项目完整复制到服务器之后，最简单就是直接运行：

```bash
bash scripts/pm2_start_lower_win.sh polymarket-lower-win .env
```

这个命令会自动完成：

- 如果 `.env` 不存在，就从 `.env.example` 复制成 `.env`
- 启动 `polymarket-lower-win-paper`
- 如果 `.env` 已经填了 Chainlink 凭证，再额外启动 `polymarket-lower-win-chainlink`
- 自动保存当前 PM2 进程列表

所以你以后在服务器上，真正常用的就是这一条命令。

## 8. 先单次检查也可以

如果你想先手动试一下单次运行，也可以：

先在服务器上做一次单次运行，确认依赖和配置没问题。

模拟盘：

```bash
bash scripts/start_paper_low_win.sh .env --once
```

如果你已经拿到 Chainlink 凭证，再检查采集器：

```bash
bash scripts/start_chainlink_collector.sh .env --max-messages 3
```

## 9. 设置开机自启

```bash
pm2 save
pm2 startup
```

`pm2 startup` 会输出一条命令，你再复制执行一次即可完成自启动注册。

## 10. 常用运维命令

查看状态：

```bash
pm2 status
```

查看日志：

```bash
pm2 logs polymarket-lower-win-paper
pm2 logs polymarket-lower-win-chainlink
```

重启：

```bash
pm2 restart polymarket-lower-win-paper
pm2 restart polymarket-lower-win-chainlink
```

停止：

```bash
pm2 stop polymarket-lower-win-paper
pm2 stop polymarket-lower-win-chainlink
```

删除进程：

```bash
pm2 delete polymarket-lower-win-paper
pm2 delete polymarket-lower-win-chainlink
```

## 11. 日志位置

项目日志默认在：

- `logs/paper_low_win/<run_id>/`
- `logs/chainlink_streams/<run_id>/`
- `logs/pm2/<start_stamp>/`

PM2 的标准输出和错误输出也会写进：

- `logs/pm2/<start_stamp>/polymarket-lower-win-paper.out.log`
- `logs/pm2/<start_stamp>/polymarket-lower-win-paper.err.log`
- `logs/pm2/<start_stamp>/polymarket-lower-win-chainlink.out.log`
- `logs/pm2/<start_stamp>/polymarket-lower-win-chainlink.err.log`

另外：

- 每次启动都会新建一个秒级目录
- 如果 `paper` 或 `chainlink collector` 跨过本地零点继续运行，会自动切到新的 `YYYYMMDD000000` 目录

## 12. 以后怎么更新代码

以后更新服务器代码，流程一般是：

```bash
cd /root/Polymarket_lower_win
git pull
. .venv/bin/activate
pip install -e .
bash scripts/pm2_start_lower_win.sh polymarket-lower-win .env
```

如果只是小改动，通常不需要重建虚拟环境。

## 当前最推荐的上线顺序

建议按这个顺序来：

1. 先把项目 clone 到云服务器
2. 先只跑 `paper` 模拟盘
3. 等 Chainlink 批准后，再补 `API key / secret`
4. 开启 `chainlink collector`
5. 后续再把结算代理逐步从 Binance/Hyperliquid 切到 Chainlink

这样最稳，也最符合当前项目的实际阶段。
